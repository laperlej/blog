Key Generation Service

Intro

The Key Generation Service(KGS) problem is about generating unique values. It can be split into 2 categories, random and sequential. Some counter problems can be conceptualised as sequential key generation and have the same solutions.

Example Problems

  • Distribute 6M free burgers among 20M users (counter problem)

Strongly Consistent Sequential Keys

Fundamentally to have a strongly consistent serial value generator there must be a singular piece of memory somewhere in the system that must be accesses sequentially with lock. This is an unavoidable bottleneck.

This type of KGS is a trap and is largely impossible to scale. Should be avoided if at all possible. If you encounter a problem with this pattern, start asking targeted clarifying questions to get some wiggle room. You must relax a requirement to make it feasible.

Any of the following would do:

  • Do not have to be perfectly ordered (distribute numbers in batches)
  • RPS is low / very low (single machine)
  • RPS is medium / low and data loss is acceptable (in-memory single machine)
